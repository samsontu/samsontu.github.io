EON Predicate Language 

Created: 2/12/96 by Samson Tu adapted from Amar Das's TLSQL language 

 Last revised: 4/7/97 


This is preliminary specification of a predicate language to be used to model expressions that may or may not refer to temporal data. Several variants languages may be defined, depending on assumption we want to make about implicit references. For example, we may want to define a variant language that allows the notion of "current time", i.e., have a predefined symbol representing the current time against which comparisons or computations can be done. 

This specification language assumes an object data model, where objects have attributes, attributes have values, and individual objects instances are organized into classes. Translation to a relational model is immediate if we assume classes map into tables and attributes into columns, and assume that attribute are single valued. However, here the single value assumption doesn't always hold. 

This specification is still incomplete. In particular, it does not allow use of functions or other common airthmetic and string operations. Also, I haven't decided exactly how to deal with the "kind-of" comparison that is common in object language, but not supported by relational language . 

Documentation on the BNF Syntax 

BNFSTART

<grain_retr>: 
        GRAIN <grain_unit> <retrieval_expr>* 
        | GRAIN <grain_unit>  <bool_expr> 
        | GRAIN <grain_unit> (AND|OR) <bool_expr> <bool_expr> ~ <bool_expr>* 
                ; 

<grain_unit> : 
                DAY | HOUR | MINUTE | SECOND | WEEK | MONTH 
        ; 

<bool_expr> : 
        '(' ~ 
                        [CONTEXT <context> (',' <context>)*] ~ 
                        [NOT] EXISTS [<range_expr>] [CONCATENATED] ~ 
                        <select_item> (',' <select_item>)* ~ 
                        IN <classname> (',' <classname>)* ~ 
                        [<when_expr>] ~ 
                        [WHERE <boolean_term> (OR <boolean_term>)*] ~ 
                        [GROUP BY <group_item> (',' <group_item>)*] ~ 
                        ')' 
                ; 

<retrieval_expr> : 
                        CONTEXT <context> (',' <context>)* ~ 
                        SELECT [<range_expr>] [CONCATENATED]~ 
                        <select_item> (',' <select_item>)* ~ 
                        [INTO {STRING}] ~ 
                        FROM <classname> (',' <classname>)* ~ 
                        [<when_expr>] ~ 
                        [WHERE <boolean_term> (OR <boolean_term>)*] ~ 
                        [GROUP BY <group_item> (',' <group_item>)*] 
            ; 

<classname> : 
                 {CLASSNAME} 
                ; 

<context>: 
                {STRING} | STAR 
                ;  

<range_expr> : 
       FIRST 
       | SECOND 
       | THIRD 
       | LAST 
       | INT_CONST {INTEGER} 
       | '(' INT_CONST {INTEGER} TO INT_CONST {INTEGER} ')' 
       ; 

<select_list> : 
       STAR 
       | <select_item> 
       | <select_list> ',' <select_item> 
       ; 

<select_item> : 
                 STAR 
       | {STRING} 
       | <aggr_expr> 
       | <attr_expr> 
       | <funct_expr> 
       | <datetime_expr> 
       ; 

<aggr_expr> : 
             {STRING} EQUAL_OP <arithmetic-op> '(' [DISTINCT] {STRING} ')' 
           | {STRING} EQUAL_OP <arithmetic-op> '(' [DISTINCT] {SLOTNAME} ')' 
           ; 
 
<arithmetic-op>: SUM | AVG | COUNT | MAX | MIN 
                ; 

<attr_expr> : 
       {STRING} EQUAL_OP {STRING} 
           | {STRING} EQUAL_OP {SLOTNAME} 
       | {STRING} EQUAL_OP <funct_expr> 
       | {STRING} EQUAL_OP <datetime_expr> 
       ; 

<funct_expr> : 
         (DATENAME|DATEPART) '(' {STRING} ',' <datetime_expr> ')' 
       | DURATION '(' <datetime_expr> (',' <datetime_expr>)* ')' 
       | <value_token> MATH_OP <value_token> 
       | <value_token> STAR <value_token> 
       ; 

<datetime_expr> : 
       TEMPATTR 
       | DATE_CONST [TIME_CONST] 
       | GETDATE '(' ')' 
       | ADDGRANULE '(' INT_CONST ',' <datetime_expr> ')' 
       ; 


<from_list> : 
       {STRING} 
       | {STRING} ',' {STRING} 
       ; 

<when_expr> : 
       WHEN <temp_comp_term> (OR <temp_comp_term>)* 
       | WHEN {STRING} [NOT] CONCURRENT_WITH {STRING} 
       ; 

<temp_comp_term> : 
       <temp_pred> (AND <temp_pred>)* 
       ; 

<temp_pred> : 
         <datetime_expr> <timepoint_op> <datetime_expr> 
       | <interval_expr> <interval_op> <interval_expr> 
       | <datetime_expr> <mixed_op> <interval_expr> 
       | DURATION '(' <datetime_expr> (',' <datetime_expr>)* ')' ~ 
                  (COMP_OP|EQUAL_OP) INT_CONST 
       | '(' <temp_comp_term> (OR <temp_comp_term>)* ')' 
       ; 

<timepoint_op> : 
         AT 
       | BEFORE 
       | AFTER 
       | ADJACENT_TO 
       ; 

<interval_op> : 
         BEFORE 
       | UNTIL 
       | LEADS 
       | STARTS 
       | EQUALS 
       | DURING 
       | FINISHES 
       | OVERLAPS 
       | SPANS 
       | LAGS 
       ; 

<interval_expr> : 
         '[' <datetime_expr> ',' <datetime_expr> ']' 
       ; 

<mixed_op> : 
         BEFORE 
       | DURING 
       | AFTER 
       ; 

<boolean_term> : 
       <boolean_factor> (AND <boolean_factor>)* 
       ; 

<boolean_factor> : 
             [NOT] <predicate> 
           | [NOT] '(' <boolean_term> (OR <boolean_term>)* ')' 
           ; 

<predicate> : 
         <value_token> COMP_OP <value_token> 
       | <value_token> EQUAL_OP <value_token> 
       | <value_token> [NOT] BETWEEN <value_token> AND <value_token> 
       | <value_token> [NOT] DBIN '(' <value_token> (',' <value_token>)*')' 
       | <value_token> [NOT] LIKE WILDCARD 
       | <value_token> IS [NOT] DBNULL 
       ; 

<value_token> : 
         {SLOTNAME} 
       | {STRING} 
       | WILDCARD 
       | {INTEGER} 
       | {REAL} 
       ; 

<group_item> : 
             {SLOTNAME} 
       | {STRING} 
       | (DATENAME|DATEPART) '(' {STRING} ',' <datetime_expr> ')' 
       ; 
BNFEND
